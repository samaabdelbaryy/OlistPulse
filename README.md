# 🛒 Retail & Web Pulse — E-Commerce Analytics Project

> A dual-dashboard analytics project examining two distinct e-commerce contexts:  
> a real Brazilian marketplace (Olist, 2016–2018) and a synthetic global storefront (2020–2025).  
> Built with Power BI and Python, structured as executive-grade performance reports.

---

## 🗂️ Repository Structure

```
├── RetailPulse_Report.pdf         # Full executive report — Olist marketplace
├── WebPulse_Report.pdf            # Full executive report — Web e-commerce
├── RetailPulse.pdf                # Power BI dashboard (4 pages)
├── WebPulse.pdf                   # Power BI dashboard (4 pages)
└── Retail_Dataset_Analysis.ipynb  # Python analysis notebook
```

## 📌 Key Findings
| Dashboard | Core Problem | Headline Miss |
|---|---|---|
| Retail Pulse | Demand-side volume gap, seller oversupply | Revenue −26.76% vs goal |
| Web Pulse | Discount addiction + retention crisis | Negative sentiment 70.69% |

---

## 🔍 Key Analysis Covered

- Revenue vs goal decomposition across 4 dashboard pages each
- Funnel analysis: session → order → customer conversion
- Delivery & logistics performance by state (Brazil)
- Customer segmentation by geography, sentiment, and age group
- Seller ecosystem health & category-level revenue per seller
- Discount architecture analysis and margin squeeze quantification
- Loyalty pipeline: At Risk / Dormant / New customer breakdown
- Cross-page causal chain synthesis with strategic recommendations
